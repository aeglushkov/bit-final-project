## Clarifications (self-Q&A)

Working through Su's reply in plain terms — kept here so future-me
re-reading these notes doesn't have to reconstruct the explanations.

### Q1. "Limited to perceiving only immediate input information and cannot go beyond current visual observations" — what does that actually mean?

Embodied VideoAgent's memory bank only contains things it has literally
seen in the input video frames. It is a perception log, not a knowledge
store — it does not pull in world knowledge from outside the video.

- If the video shows a kitchen with a fridge but the agent never sees
  the *inside* of the fridge, it cannot reason "fridges usually contain
  food, so there's probably a milk carton in there."
- A human would. The VLM itself *knows* this from training, but the
  agent's memory bank does not tap into that prior — it only stores
  observed facts.

### Q2. "An agent that fully leverages long-term memory acquired during continual learning will be the key direction for future agents" — what does Su mean and how is that different from current agents?

Today's agents (Embodied VideoAgent included) are essentially
single-episode: hand them one video → they build a memory bank for
that video → at the end the memory is thrown away. Next video, start
from zero.

Su wants an agent that carries knowledge *across* episodes — every
video adds to a permanent, growing knowledge store, closer to lived
experience than to amnesia.

- *Current:* Walks into a new house. "I see a sofa, a TV, a coffee
  table." End. Knows nothing else.
- *Long-term-memory agent:* Walks into a new house. "I see a sofa, a
  TV, a coffee table — and from all the living rooms I've seen
  before, I know living rooms usually connect to a hallway, the sofa
  usually faces the TV, and there's typically a bedroom on the same
  floor."

### Q3. "Building brain-like contextual memory on top of Embodied VideoAgent to fully unleash the model's learned knowledge will be critical to our research" — what does Su mean and how would it actually be used?

Two ideas combined:

- *Brain-like / contextual* = associative. In the brain, recalling
  one thing drags related things along with it (smell of pine →
  Christmas tree → childhood living room → grandfather's chair). One
  cue brings the whole context back.
- *On top of Embodied VideoAgent* = keep the object memory bank as
  the foundation but add a higher layer that links objects into
  scenes and scenes into broader context.

How it gets used in practice: store scene-level entries
(representative frame + all items in that scene). At query time, use
a generation model to "complete" the scene — fill in what's likely
there even if not directly observed. Cross-match the completed scene
against memory and reason over it. The VLM's *learned* knowledge and
the agent's *observed* memory work together, instead of the agent
ignoring everything the VLM already knows.

### Q4. "It only sparsely records isolated objects and individual actions, yet fails to form targeted memory for holistic scenes, which explains its unsatisfactory performance on tasks like OpenEQA" — why?

**Structural problem:** Embodied VideoAgent's memory is per-object.
Each entry is one object with its 3D box, state, features. No entry
says "this is the kitchen scene." Scenes exist only implicitly as a
bag of object entries.

OpenEQA asks *holistic* questions ("What was the layout of the room I
walked through?", "What kind of room is this?") where the answer is
about the whole scene, not one object. A bag-of-objects memory is
the wrong shape for this.

**Numbers** (from `literature/EmbodiedVideoAgent/analysis.md` §5.2):

| OpenEQA full set         | ALL  |
|--------------------------|-----:|
| GPT-4 + LLaVA-1.5 caps   | 43.6 |
| Video-LLaVA (end-to-end) | 39.2 |
| GPT-4 + ConceptGraphs    | 36.5 |
| LLaMA-VID                | 33.6 |

| OpenEQA hard subset        | ALL  |
|----------------------------|-----:|
| E-VideoAgent (GPT-4o)      | 47.0 |
| E-VideoAgent (InternVL2-8B)| 41.2 |
| VideoAgent                 | 36.3 |
| Video-LLaVA                | 30.6 |
| LLaMA-VID                  | 29.4 |

On the hard subset Embodied VideoAgent wins (47.0 vs 36.3 for
VideoAgent), but absolute is still ~47 — i.e. it gets more than half
wrong. On the *full* set the paper doesn't even report numbers; a
much simpler method (GPT-4 reading frame captions) already gets
43.6. Translation: all the 3D-object-memory machinery is not paying
off proportionally on holistic scene questions. That gap is what Su
calls "unsatisfactory."

### Q5. The worked example: "I am sitting on the sofa in the living room and watching TV. Which room is behind me?"

Setup: input is a video of someone in a living room watching TV. The
text query is the question above.

Why this is hard for current Embodied VideoAgent:

- Memory has `sofa#1`, `tv#1`, `coffee_table#1` with 3D positions.
- "Behind me" requires an *egocentric* viewpoint — the asker's body
  orientation. The agent has world coordinates but no notion of
  where the asker is sitting and which way they are facing.
- Even with the viewpoint, "the room behind" needs room-level
  structure. The memory has no rooms, just objects.

Su's proposed flow:

1. *Anchor the viewpoint.* "Sitting on sofa watching TV" → viewer is
   on the sofa, oriented toward the TV.
2. *Use a generation model to reconstruct.* Feed key frames + the
   question into a scene generator. It produces a plausible scene
   including what is behind the viewpoint, using both world-knowledge
   priors and what has been seen in this specific video.
3. *Cross-match against memory.* If the bank has frames showing a
   kitchen door behind the sofa from a different angle, that match
   wins.
4. *Answer.* "The kitchen is behind you."

The trick: when memory alone does not have the answer, the
generation model fills in a hypothesis using learned world
knowledge, and the memory bank confirms or refines it. Memory +
generative imagination working together — that is the "brain-like"
part.

  - Both are needed; the agent's job is to orchestrate them.

## Su's proposal — scene-level contextual memory on top of Embodied VideoAgent

- **Memory structure:** mapping from *representative scene frames* → all
  items / objects in that scene (extension of the current per-object index
  upward to the scene level).
- **At query time:**
  1. Sample key frames from the input video.
  2. Feed key frames + the textual query into a *generation model* as
     conditional inputs to "reconstruct analogous scene layouts" — i.e.
     contextual recollection.
  3. Use the *generated* scene features to cross-match and retrieve from the
     memory bank.
  4. Reason over the retrieved scene + objects to answer.
- **Worked example (Su's):** "I am sitting on the sofa in the living room
  and watching TV. Which room is behind me?"
  - Stored scene reps + query → generation model fills in the recalled
    layout (living room with sofa + TV, plus what's behind the viewpoint).
  - Generated scene features → retrieval + reasoning → answer.
- **Framing claim:** contextual memory is the *core ability* that lets an
  agent accumulate and reuse acquired world knowledge — i.e. this is
  positioned as the central research bet, not a side feature.
